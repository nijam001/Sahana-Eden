# WIF3005 Software Maintenance and Evolution

## Group Project Description
**Semester 1 2024/2025**

---

## Group Project Objective

To maintain, evolve, collaborate, test build, and deploy your selected code using GitHub

---

## Project Activities and Tasks

1. Create a new project with a repository. It can be either cloned from an existing project or developed from scratch.
2. Collaborate as a team, each member taking on different roles.
3. Analyze the selected code.
4. Identify components requiring maintenance and evolution where applicable.
5. Conduct an impact analysis before implementing any changes.
6. Assign tasks and responsibilities by the team leader.
7. The team leader should review and approve code changes.
8. Continuously build and test the code on the preferred branch.
9. Continuously deploy the application to the chosen environment.

---

## Group Formation and Leadership

- Form a group of 5 to 7 members.
- Elect a leader for your group to coordinate the teamwork. The group leader might change every two weeks to share responsibilities.

The report must include:

1. Cover page with group members' details
